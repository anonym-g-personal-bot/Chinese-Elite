你必须严格遵循以下JSON格式输出分析结果。你的输出必须是，且仅是一个符合 RFC 8259 标准的、不带任何注释或代码块标记 (```json) 的、单一完整的JSON对象。

**核心提取规则 (必须严格遵守):**

1.  **【ID与名称规范 - 最重要】**:
    *   `id` 字段是实体的**唯一标识符**，必须使用Markdown链接的目标页面标题。例如，在 `[[中华人民共和国|中国]]` 中，`id` 必须是 `"中华人民共和国"`。
    *   `name.zh-cn` 字段是一个**名称列表**。它应该包含实体所有已知的中文名称，包括 `id` 本身 (首项) 和Markdown链接中显示的文本。
    *   **列表的第一个元素**应该是 id 本身。
    *   示例: 对于 `[[中华人民共和国中央人民政府 (1949年—1954年)|中央人民政府]]`，正确的节点结构是：
        `"id": "中华人民共和国中央人民政府 (1949年—1954年)"`
        `"name": { "zh-cn": ["中华人民共和国中央人民政府 (1949年—1954年)", "中央人民政府", ], "en": [] }`

2.  **【关系类型与节点类型约束】**: 关系的方向为主语->宾语 (source->target)。必须严格遵守以下类型约束：
    *   `BORN_IN`: Person -> Location (必须是唯一的、最精确的出生地，如市或县)
    *   `ALUMNUS_OF`: Person -> Organization
    *   `MEMBER_OF`: Person/Organization -> Organization
    *   `CHILD_OF`: Person -> Person (source是子女, target是父母)
    *   `SPOUSE_OF`: Person -> Person
    *   `SIBLING_OF`: Person -> Person
    *   `LOVER_OF`: Person -> Person
    *   `RELATIVE_OF`: Person -> Person (用于除直系亲属、配偶、情人外的其他亲戚)
    *   `FRIEND_OF`: Person/Organization -> Person/Organization (私人友谊或密切合作)
    *   `ENEMY_OF`: Person/Organization -> Person/Organization (敌对或对立关系)
    *   `SUBORDINATE_OF`: Person/Organization -> Person/Organization (source级别低于target)
    *   `MET_WITH`: Person -> Person (正式或非正式会面)
    *   `FOUNDED`: Person/Organization -> Organization/Movement
    *   `PUSHED`: Person/Organization/Event/Movement/Document -> Person/Organization/Event/Movement/Document (起推动、促进作用)
    *   `BLOCKED`: Person/Organization/Event/Movement/Document -> Person/Organization/Event/Movement/Document (起阻碍、反对作用)
    *   `INFLUENCED`: Person/Organization/Event/Movement/Document -> Person/Organization/Event/Movement/Document/Location (通用影响关系，当其他类型不适用时使用，严禁滥用)

3.  **【精确性优先】**: 只提取文本中明确提及或具有强上下文支撑的关系。禁止基于常识进行过度推断。例如，不要仅仅因为两人在同一组织就推断他们是`FRIEND_OF`。

4.  **【时间信息】**: 对于可能随时间变化的关系 (如 `FRIEND_OF`, `ENEMY_OF`, `LOVER_OF`)，如果文本提及多个不连续的时间段，请使用数组记录所有时间段。如果只有一个时间点，则使用单个字符串。

5.  **【节点ID规范】**: `relationships` 中的 `source` 和 `target` 必须与 `nodes` 列表中的 `id` 完全匹配。所有在关系中提及的实体都必须在 `nodes` 列表中有对应的条目。

6.  **【避免类型混淆】**: 严禁创造新的实体或关系类型。严禁在不当的节点类型组合之间使用关系。例如，邓小平 **推动了 (PUSHED)** 改革开放 (Movement)，而不能是 **创立 (FOUNDED)** 改革开放。
